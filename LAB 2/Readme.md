The process begins with an in-depth data exploration phase. The script loads the dataset and immediately checks its core structure—such as its shape, data types, and basic statistics. It also scans for potential issues like missing values and duplicate records, while checking how the target classes are distributed. Once the baseline is established, the script shifts to data visualization, automatically generating histograms to show individual feature distributions and a correlation heatmap to uncover linear relationships between different variables.

Data is rarely perfect, which is why the script includes a thorough data cleaning routine. It automatically handles missing data by filling numerical columns with their median and categorical columns with their mode, while also dropping any duplicate rows. To ensure complete transparency, it prints a side-by-side comparison of the mean and median before and after the cleaning process, allowing us to see exactly how the data changed. Finally, the pipeline wraps up with feature engineering, applying label encoding and one-hot encoding to categorical features so that the final output is fully optimized for downstream analysis.
